In a later section we discuss running a Monte Carlo simulation where theerror in the \nhi\ is propagated through our final results, thus we requireknowledge of the uncertainty of the velocity centers and velocity widths.There are two ways we approach deriving in the velocity center and velocitywidth of the \hi\ integration range, one parametric, one non-parametric.For both methods we first take the marginal distributions for eachparameter by finding the combination of parameters corresponding to themaximum correlation, then assign the marginal distribution for oneparameter as the correlations for which the other parameter is heldconstant at the corresponding maximum. Put another way, we found theparameter values corresponding to the maximum correlation coefficient, heldone parameter constant at the maximum, and stored all of the correlationcoefficients as a function of the other parameter. Essentially we had thecorrelation coefficient value as a function of each parameter. We usedthese functions to create a probability density function for each parameter.The first method to determine errors on the parameters is to lower athreshold value until some fraction of the area in the marginaldistribution is equal to a given confidence value, e.g. 95\%. Thecorresponding parameter values at the edges of the threshold represent theerror on about the maximum likelihood estimate, the parameter value at thepeak correlation value. We can then use a skewed Gaussian distribution toretrieve random variable samples for the Monte Carlo simulation.The second method is to create probability density functions (PDFs) foreach parameter allowing for random sampling of each parameter during theMonte Carlo. We consider the correlation coefficients to trace out the PDFsby treating the correlation coefficients for each velocity as the relativenumber of counts in velocity bins. We normalize the 1D histogram for thevelocity parameters to have an area of 1, thus creating a probabilitydistribution function. These non-parametric PDFs allow us to generatediscrete random variables for the Monte Carlo simulation.We choose to use the non-parametric approach for determining the error onthe \hi\ velocity range given that the \nhi\ is non-linearly dependent onthe \hi\ velocity range. Adopting a specific distribution from which tosample random variables in our Monte Carlo simulation means that the \nhi\depends linearly on the \hi\ velocity range. To check that we constructedthe PDFs correctly with the non-parametric method we took 10,000 randomsamples from the velocity center and width parameter PDFs, and compare theresulting distribution to the marginal distributions of correlationcoefficients. The results are in Figure~\ref{fig:pdfs}.,V

    We derive \nhtwo\ in a maximum likelihood framework. We first assume a    molecular cloud has a single Dust-to-Gas Ratio (DGR) and a single velocity    range which traces only atomic gas associated with the cloud. Thus the dust    column will trace the gas column in the following way    \begin{equation}        A_V = {\rm DGR}\times [2 N({\rm H}_2) + N({\rm H}\,\textsc{i})]    \end{equation}    \noindent and in the case where there is no molecular hydrogen along the    column, the dust column simply traces the \hi\ column scaled by the    DGR\@.  Our task then follows to estimate the DGR and \nhi\ within    strictly atomic columns of gas, and then scale $A_V$ with the DGR to    estimate the total gas column, and subtract off the \hi\ component.    % ==========================================================================    \subsection{Isolating Atomic Regions}                We wish to exclude regions in our analysis with molecular gas so that        the dust column is only tracing atomic gas. We follow the methods of        \citet{planck11_24} to include only atomic regions for deriving the DGR        of each cloud. For a cloud where infrared emission traces dust which        only traces atomic gas, the \hi\ emission should correlate with the        dust emission to within the error, assuming optically thin \hi. The        $A_V$ map should trace the dust column density which will trace the        \hi\ column density. In this case, our model $A_V$ as a function of        position, $A_{V,{\rm model}}$ is written as        \begin{equation}            A_{V,{\rm model}}(x,y) =             {\rm DGR} \times N_{\rm HI}(x,y) + R(x,y) + Z        \end{equation}        \noindent where $N_{\rm HI}(x,y)$ is the \hi\ column density as a        function of position, $R(x,y)$ represents the contribution of noise and        correlated residuals between the model and observed $A_V$, and $Z$ is a        zero point, or an intercept for the map. $R(x,y)$ accounts for any        extinction not correlated with \nhi, including emission from the cosmic        infrared background, and potential dust emission tracing molecular or        ionized gas.        We solve for the DGR by performing a linear least squares solution                \begin{equation}\label{eq:dgr}        DGR = {(\bm{A}^T \bm{A})}^{-1} \times (\bm{A}^T \bm{b})        \end{equation}        \noindent where $\bm{b}$ is is a vector of the Planck $A_V$ data points        divided by their respective error and $\bm{A}$ is a vector including        the \nhi\ data points divided by the respective error of the $A_V$. We        use the pseudo-inverse to solve this equation.        An ideal model would reveal residuals, the difference between the data        and the model $A_V$, which consisted only of white noise. However        previous work on the correlation between dust and gas \citep[e.g.,        ][]{planck14_17} has shown that there is excess dust emission not        associated with a galactic cloud, i.e.\ the cosmic infrared and        microwave backgrounds, and galactic emission representing dust not        associated with \hi\ such as dust associated with \htwo\ and \hii\ gas,        synchrotron emission, and free-free emission. How each of these        components may contribute to the derived optical extinction is        complicated, we discuss this effect in        Section\,\ref{sec:residuals_discussion}.        Our initial interest in using this model is to exclude all excess dust        emission not correlated with \hi\ listed above. We adopt the iterative        approach of \citet{planck11_24} where we solve for the DGR using        Equation~\ref{eq:dgr} including all pixels in the map. Next we examine        the probability density function (PDF) of the residuals, the        data---model difference, where the excess dust emission (excess $A_V$)        will show as high valued positive residuals. Assuming Gaussian        distributed noise in the $A_V$ map, we will expect the negative values        of the residuals to follow a Gaussian distribution. We fit the rising        side of the residual PDF where $R < 0$\,mag with a Gaussian, and        exclude all pixels greater than $3\,\times\,\sigma$ of the fitted        Gaussian. We then recalculate the DGR with the new mask and repeat        until the derived DGR converges.        This method requires a priori knowledge of the \hi\ velocity        integration range in order to calculate \nhi. The CO line width for        each cloud is on order of a few km/s, and we expect the \hi\ line width        to be on order of $5\,\times$ the line width of the CO        \citep{andersson91}. The effects of choosing a different line width and        the scaling of the fitted residual width are discussed in        Section~\ref{sec:masking_discussion}.        % Changing the residual masking parameters in Perseus        % number of pixels used in analysis after changing the following        % Resid scale    HI range    npix   DGR     HI width        % 2.0            -10, 10     8627   0.08    16        % 3.0            -10, 10     10231  0.09    14        % 4.0            -10, 10     10424  0.09    14        % 3.0            -20, 20     9760   0.08    17        % 3.0            -5, 5       10915  0.09    14        % 3.0            -40, 20     8498   0.08    16    % ==========================================================================    \subsection{Overview of Maximum Likelihood Estimation}        Given Gaussian-distributed noise we can derive the parameters of the        maximum likelihood estimate by searching over parameter space. Given        observations, or pixels $x_1, \dots, x_N$ where $N$ is the total number        of elements or pixels, the probability of a model reproducing the        observations is given by\\                \begin{equation}             \mathcal{L} = p(x_1,\dots,x_N) =              \prod_{n=1}^N \frac{1}{\sqrt{2\pi}\sigma}            \exp{\left[\frac{-{(x_{n{\rm, data}} -             x_{n{\rm, model}})}^2}{2\sigma^2}\right]}        \end{equation}        \noindent where $\sigma^2$ is the variance of the data, $x_{n{\rm,        data}}$ are the data, and $x_{n{\rm, model}}$ are the model elements.        This corresponds to a log-likelihood of        \begin{equation}            \log(\mathcal{L}) = -\frac{1}{2}N{\rm \log}(2\pi\sigma^2) -            \sum_{n=1}^N \frac{{(x_{n{\rm, data}} -             x_{n{\rm, model}})}^2}{2\sigma^2}        \end{equation}        \noindent where the likelihood and the log-likelihoods are maximized        for the same maximum likelihood estimate. In a numerical grid search of        the likelihood space across different parameters, it is more practical        to exclude constants in the likelihood or log-likelihood calculations,        and normalize after the full parameter space is explored. The        log-likelihood calculation we make is        \begin{equation}\label{eq:logL}            \log(\mathcal{L}) = - \sum_{n=1}^N \frac{{(x_{n{\rm, data}} -             x_{n{\rm, model}})}^2}{2\sigma^2}        \end{equation}        After the full grid search which we assume includes parameter space        which fully samples the true likelihood space, we then normalize in        log-likelihood space by subtracting the maximum log-likelihood from all        log-likelihoods. We then covert the log-likelihoods to likelihoods and        perform an additional normalization where we divide the likelihoods by        the sum of all likelihoods. This ensures that the sum of all        likelihoods is 1, which is necessary for a fully-sampled likelihood        space.        % ==========================================================================    \subsection{Application of Maximum Likelihood Estimation}        There are several assumptions in our model which likely underestimate        the true error in our derivation of the DGR and the \hi\ integration        width. First, the DGR is likely to vary within a factor of 2 across the        cloud, second, the Planck $A_V$ errors may be underestimated due to        systematics. We can account for this affect by adopting the method        proposed by Chapter $15$ in Numerical Recipes. We can perform an        iterative approach, whereby we derive the best-fit model using the MLE        grid search, and subsequently calculate the uncertainty between the        best-fit model and the data given by the variance between the        model and the data        \begin{equation}            \sigma^2 = \sum_{n=1}^N                 \frac{{[x_{n{\rm, data}} - x_{n{\rm, model}}]}^2}{\nu}        \end{equation}        \noindent where $\nu$ is the number of degrees of freedom. With this        new estimation of the variance we can rerun our likelihood        calculations to produce a more realistic calculation of the variances        and covariance of \hi\ integration width and the DGR\@. Upon our        first iteration, where we use only the Planck $A_V$ map errors to        estimate likelihoods, we find that the model $A_V$ map deviates        significantly from the Planck $A_V$ map. See        Figure~\ref{fig:avmod_vs_av} for a plot of the model $A_V$ vs.\ the        observed $A_V$. We find that the estimated standard deviation between        the model and the Planck $A_V$, $\sigma$ = 0.22 mag.
